### G4 아키텍처/상태/롤백 관점 구조 검토 보고서 (한결)

#### 1. editor↔roughcut sync & NLE commit boundary 설계
- **격리된 샌드박스 상태**: 러프컷 조립, 순서 변경, 분할/병합 등 모든 편집 액션은 `Roughcut state` 내의 격리 버퍼(Sandbox)에만 머무르며, 메인 타임라인(`Editor state`)을 직접 수정하지 않습니다.
- **Commit Boundary 규정**: 사용자가 '커밋(Commit/Apply)'을 실행할 때만 자막 타임스탬프 계산기가 실행되어 메인 타임라인으로 데이터를 병합합니다. 병합 연산 도중 타임라인 오버랩 등의 충돌 검출 시, 전체 연산을 트랜잭션 단위로 즉시 롤백(Rollback)하여 원본 자막 데이터 파손을 원천 차단합니다.

#### 2. Three-domain Save 및 마이그레이션 호환성
- **3대 도메인 영속성 격리**: `Editor state` (원본 자막/타임라인), `Roughcut state` (연습노트 후보군/노드 좌표/스토리라인), `Shortform state` (숏폼 프레이밍 및 핸드오프 리스트)의 3대 상태 도메인을 별개의 데이터 구조 클래스로 엄격히 추상화 분리합니다.
- **일괄 영속화 (Three-domain save)**: 하나의 프로젝트 `.json` 파일 저장 시, 3대 도메인의 직렬화 데이터를 하나의 루트 사전(dictionary) 객체로 패키징하여 단일 파일 트랜잭션 쓰기를 수행합니다. 구버전 로드 시에는 스키마 마이그레이션 가드(Schema validator)를 거쳐 결손 메타데이터를 안전하게 기본 상태로 무해화(Null-safe parsing)합니다.

#### 3. B-roll 레이어 및 대체 테이크(Alternate take) 추상화
- **메타데이터 트랙화**: NLE 구조 내에 B-roll 및 인서트 샷 레이어링을 담을 세그먼트 채널(Segment Channel) 개념을 구현하여, 주 비디오 데이터를 변경하지 않고 메타데이터 상에서만 비디오 재생 좌표와 자막 오버레이를 오버라이드합니다.
- **비파괴적 오프셋 적용**: 대체 촬영본(Alternate take) 스위칭 시, 원본 자막의 타임코드를 직접 건드리지 않고 오직 각 테이크별 오디오 싱크 시간 차이(ms Offset)를 노드 메타데이터에 등록하여, 프리뷰 재생 시점에만 오프셋을 실시간 계산 적용하는 비파괴적 정렬 구조를 갖춥니다.

#### 4. Vertical Crop 및 숏폼 핸드오프 페이로드
- **느슨한 데이터 바인딩**: 숏폼 전송 시, 9:16 화면 크롭 정보를 담은 vertical crop (X/Y 좌표 오프셋 및 스케일) 메타데이터와 클립 시간 범위, 자막 텍스트 싱크 인덱스를 패키징한 Handoff Payload 객체를 브릿지 API를 통해 전송하며, 숏폼 모듈 내부 로직과는 철저히 느슨한 결합(Loose coupling) 인터페이스를 유지합니다.
