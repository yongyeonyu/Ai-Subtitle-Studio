### G4 편집자 워크플로우/인지도 관점 검토 보고서 (유진)

#### 1. 첫 진입 및 시드 구분 워크플로우 (Initial Seed Provenance)
- **헷갈릴 포인트**: 러프컷 진입 시 카드 배열이 왜 이렇게 되어 있는지 출처를 몰라 혼란스러울 수 있습니다.
- **워크플로우 보완**: 캔버스 타이틀에 현재 정렬 상태가 `LLM 추천 순서`인지 `타임라인 기본 순서`인지 명확한 활성 라벨(Active Label)을 상시 표출하고, 노드 카드 상단에도 순서 출처 정보(예: "LLM", "Timeline")를 컴팩트한 태그로 노출하여 인지성을 확보합니다.

#### 2. 다중 연습노트(A/B) 작업 시 상태 혼선
- **헷갈릴 포인트**: 사용자가 복수의 조립 후보군(연습노트)을 전환하며 작업할 때, 현재 어떤 후보(A or B)의 캔버스에서 작업 중인지 혼동하여 엄뚱한 후보에 수동 배치를 덮어쓰는 오류가 생길 수 있습니다.
- **워크플로우 보완**: 캔버스 테두리에 활성화된 연습노트의 성격에 맞춘 2D 브랜딩 보더 및 타이틀 바 내에 `연습노트 B 편집 중` 등의 상태 메세지를 직관적으로 표시하고, 다른 연습노트 전환 시 변경 사항이 임시 자동 저장(Auto-commit)됨을 인지시키는 미세 가이드를 노출합니다.

#### 3. 분할/병합(Split/Merge) 조작 피드백 부족
- **헷갈릴 포인트**: 드래그앤드롭으로 카드를 겹쳐 합칠 때(Merge) 마우스를 떼는 지점이 모호하여 원치 않는 순서 재배치로 오인되거나, 컷 분할 시 경계면 컷이 정상적으로 쪼개졌는지 시각적 피드백이 약해 조작을 반복할 수 있습니다.
- **워크플로우 보완**:
  * 병합 드래그 도중 노드가 겹쳐지면 대상 노드 보더가 점선으로 깜빡이며 `병합 대상`임을 시각적 2D 하이라이트로 미리 보여줍니다(Hover-overlay).
  * 분할 완료 시, 쪼개진 2개의 카드가 캔버스 상에서 미세하게 좌우로 흔들리거나 분리되는 2D Micro-animation 피드백을 주어 성공을 인지시킵니다.

#### 4. 숏폼 전송 60초 버짓 실시간 게이지 미비
- **헷갈릴 포인트**: 숏폼 전송을 시도하기 전까지는 내가 고른 세그먼트들의 총합이 60초를 넘었는지 알 수 없어, 전송 클릭 시점에 60초 초과 차단 팝업을 받으면 편집 흐름이 끊겨 답답함을 느낍니다.
- **워크플로우 보완**: 숏폼 바구니 패널 하단에 실시간 연장 시간 게이지 바(예: `현재 54초 / 60초`)를 항상 노출하고, 58초를 초과하면 게이지 바 색상이 노란색, 60초를 초과하면 빨간색으로 변경되어 전송 불가 상태임을 직관적으로 사전 안내합니다.
