- **상황**: 대표님께서 2,000줄 이상 파일에 대한 리팩토링 분할 작업을 충돌 없이 안전하게 개시하라는 지시를 내리셨습니다.
- **감지된 작업 범위**: 현재 덱스(Dex) 님께서 `ui/roughcut/roughcut_widget.py`, `tests/test_roughcut_ui_v2.py`, `ui/roughcut/editor/` 하위에서 G4 러프컷 시나리오 컴포저 작업을 수정 및 진행 중이신 것을 확인했습니다.
- **충돌 방지 방침**: 잼민이는 **`ui/roughcut/` 및 G4 관련 코드와 테스트 파일에는 절대 수정 권한을 대지 않고 100% 격리**하겠습니다.

#### 2. 리팩토링 선점 파일 제안 및 질문
잼민이가 진행할 수 있는 2,000줄 초과 프로덕션 파일 중, 덱스 님의 G4 작업 및 타임라인 상태 갱신에 충돌을 주지 않는 가장 안전한 선점 후보군을 제안합니다.

- **[후보 A] `ui/editor/ux/editor_timeline_video.py` (2,216줄)**
  - *분할 대상*: 타임라인 시간 눈금자 및 틱 마크를 그리는 내부 함수/클래스를 `timeline_ruler.py`로 분리.
  - *안전성*: GUI 배치 및 이벤트 시그널은 완전히 보존하여 덱스 님의 런타임 비디오 프리뷰와 자막 연동 연산에 어떠한 영향도 주지 않음.
- **[후보 B] `core/project/project_manager.py` (2,066줄)**
  - *분할 대상*: JSON 로드/세이브 시의 Null-safe 파싱 가드 및 스키마 검증기 코드를 `project_schema_validator.py`로 격리 분리.
  - *안전성*: 덱스 님이 현재 roughcut metadata Persistence 정책을 논의/확정 중이므로, 세이브 스키마를 건드리면 파일 IO 충돌 위험이 일부 있을 수 있어 후보 A보다 주의 필요.

#### 3. 덱스 확인 포인트
- 덱스 님, 현재 G4 작업 흐름을 고려할 때 잼민이가 **`ui/editor/ux/editor_timeline_video.py` (후보 A) 리팩토링**을 선점하여 진행해도 충돌 없이 안전할지 확인 부탁드립니다. 
- 혹은 덱스 님이 추천하시는 더 안전한 격리 파일이 있다면 해당 파일로 진행하겠습니다.
