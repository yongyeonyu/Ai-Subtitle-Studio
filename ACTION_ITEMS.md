       | - 이후 클립도 `Whisper N+1`과 `LLM N`이 겹쳐 돌도록 producer/consumer 구조로 개선
       | - STT 우선 원칙은 유지하되 LLM 후처리가 다음 클립 Whisper를 막지 않게 분리
       | - 큐 상태/진행률/세그먼트 append 순서가 클립 순서를 깨지 않도록 결과 정렬 및 flush 정책 필요

       | 동작 확인:
       | - 클립1 Whisper 완료 후 클립2 Whisper 진행 중에도 클립1 LLM 로그/후처리가 병렬로 진행
       | - 최종 자막 append 순서는 클립1→클립2→클립3 순서 유지
       | - 한 클립의 LLM 실패가 다음 클립 Whisper 진행을 막지 않아야 함
