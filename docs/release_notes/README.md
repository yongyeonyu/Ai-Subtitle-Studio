# Release Notes

This folder owns source-app release checkpoint notes.

Canonical files:

- `RELEASE_v04.00.07.md` and newer
- `../PROJECT_STATE.md`
- `../VALIDATION.md`
- `../quality_validation/test_result.md`

Rules:

- Keep `RELEASE_v04.00.07.md` and newer only unless the owner explicitly changes retention policy.
- Do not use release notes as a substitute for current validation output.
- Update release notes only for release-relevant, verified behavior or packaging changes.
- Historical root-path mentions inside old release notes are preserved as history unless they block current navigation.
