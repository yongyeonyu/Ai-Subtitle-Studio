# Product Behavior

This folder points to user-visible behavior, App Store readiness, UI/UX
guardrails, and product policy.

Canonical files:

- `../APP_STORE_SUBMISSION_READINESS.md`
- `../PROJECT_STATE.md`
- `../project_reference/PRODUCT_README.md`
- `../../AGENTS.md`

Rules:

- Do not change UI, UX, labels, layout, colors, shortcuts, menus, or popup behavior unless the owner explicitly asks.
- Do not claim App Store readiness from source-app QA alone.
- Keep product claims aligned with current code-backed behavior and release notes.
