# RELEASE v03.03.00

## 핵심 요약
- v03.02.01부터 v03.02.18까지의 PHASE2 작업을 v03.03.00 릴리즈로 고정했습니다.
- AI 설정을 탭 구조로 정리하고, 에디터/러프컷 LLM 설정을 분리했습니다.
- 자막 생성 중 에디터에서 러프컷 중분류 초안을 즉시 만들고 A/B/C 레인으로 표시합니다.
- 사이드바, 터미널 로그, 타임라인, 저장 상태, 자동 처리 UI 흐름을 안정화했습니다.
- 백그라운드 파이프라인이 삭제된 PyQt MainWindow에 접근해 발생하던 종료 오류를 방지했습니다.

## AI 설정
- AI 설정창을 `빠른 설정`, `에디터 LLM`, `러프컷 LLM`, `AI` 탭으로 정리했습니다.
- 기존 설정의 LLM 프롬프트와 LLM 처리 스레드를 에디터 LLM 영역으로 이동하고, 러프컷 LLM 처리 스레드를 별도로 분리했습니다.
- 기존 `설정 > 자막 품질` 항목을 AI 설정 안으로 이동했습니다.
- 빠른 인식 프리셋에서는 에디터 러프컷 초안 생성 옵션이 자동으로 꺼지도록 했습니다.

## 에디터 러프컷 초안
- `core/roughcut/editor_draft.py`를 추가해 에디터 자막 생성 중 러프컷 초안 후보를 만들고 `roughcut_state`에 공유 저장합니다.
- 자막 생성 중 로컬 규칙으로 A/B/C 중분류 초안을 먼저 즉시 생성하고, LLM 응답이 돌아오면 같은 후보를 refinement 결과로 갱신합니다.
- 에디터 타임라인에 러프컷 중분류 A/B/C 레인을 추가했습니다.
- 러프컷 에디터는 프로젝트의 공유 `roughcut_state`를 통해 에디터에서 만든 후보를 복원합니다.
- 멀티클립 source path와 clip boundary metadata를 후보 payload에 보존합니다.

## 타임라인 / 에디터 UI
- 자막 에디터 선택 시 캔버스와 동일한 노란색 focus border를 표시합니다.
- 자막 줄 또는 타임라인 세그먼트를 선택하면 플레이헤드와 비디오 위치가 함께 이동합니다.
- 긴 타임라인에서 글로벌 캔버스 `ㅁ` 화면 맞춤이 확대 기본값으로 튀지 않도록 보정했습니다.
- `음성`과 `무음` 텍스트 세그먼트를 초록/주황 계열로 구분했습니다.
- 자막/화자/음성감지/분석 레인을 글로벌 캔버스 쪽으로 정렬하고 하단 내부 여백을 제거했습니다.
- 시작 버튼에 뇌 이모지가 잠깐 표시되던 초기 라벨을 제거했습니다.

## 사이드바 / 상태 / 자동 처리
- 터미널 로그를 사이드바 하단으로 통합했습니다.
- 하단 메뉴의 `터미널` 버튼을 `사이드바` 버튼으로 변경하고, 프로젝트 사이드바 숨김/표시 기능을 통합했습니다.
- 사이드바 안에 있던 하단 메뉴 아이콘을 작업영역 하단으로 이동하고 창 폭에 맞게 축소되도록 했습니다.
- iCloud/NAS 자동 처리 카운트를 `대기 : 영상 00개 / 음성 00개`, `완료 : 영상 00개 / 음성 00개` 형식으로 정리했습니다.
- 자막 생성 완료 상태를 좌측 상태, 터미널, 큐 테이블과 동기화했습니다.
- 저장 후에도 dirty 상태가 빨간색으로 남고 종료 시 저장 확인이 뜨던 문제를 실제 세그먼트 변경 기준으로 보정했습니다.

## 안정성
- 홈 화면 재구성 중 삭제된 `TerminalLogWidget`을 재사용해 발생하던 RuntimeError를 방지했습니다.
- 단일 파일 파이프라인에서 백그라운드 optimizer가 삭제된 `MainWindow` 시그널에 접근해 발생하던 `wrapped C/C++ object ... has been deleted` 치명 오류를 guard했습니다.
- PyQt 객체 생존 여부를 확인하는 단위 테스트를 추가했습니다.
- LLM 러프컷 초안 생성이 timeout될 경우 로컬 초안을 유지하고, 짧은 cooldown 뒤 재시도하도록 했습니다.

## 문서 / 운영 규칙
- 릴리즈 시 middle 버전을 증가시키고 새 릴리즈 노트 파일을 만드는 규칙을 문서화했습니다.
- routine 기능 수정에서는 꼭 필요한 검증만 수행하고, 문서/전체 검증은 릴리즈 때 일괄 수행하는 규칙을 추가했습니다.
- 리팩토링은 대표님이 명시 요청할 때만 실행하며, 사용하지 않는 것으로 의심되는 파일은 삭제 전 `.codex_work/`로 이동해 보관하는 규칙을 추가했습니다.
- `File_structure.txt`의 설명 공백을 넓혀 가독성을 개선했습니다.
