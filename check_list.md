## 사용 원칙

- 이 문서는 대표님이 직접 앱을 실행해서 확인해야 하는 항목만 정리합니다.
- 구현이 완료된 액션아이템도 실제 UX 확인이 필요하면 이 문서에 남깁니다.
- 확인 완료된 항목은 체크 후 다음 정리 때 제거합니다.
- 새 구현 항목은 `ACTION_ITEMS.md`에 먼저 적고, 대표님 확인이 필요한 UX/동작만 이 문서로 옮깁니다.
- 문서 운영 원칙은 `ACTION_ITEMS.md` 상단 기준 영역과 `AGENTS.md`에서 관리합니다.

## 대표님 확인 체크포인트 테스트

- [ ] CP-12 | 멀티클립 전환/저장/재생
  - 단일클립에서 멀티클립으로 전환 가능해야 함
  - 멀티클립에서 러프컷으로 이동해도 단일/멀티 편집 상태와 러프컷 상태가 분리되어야 함

- [ ] CP-13 | STT 모드

  - 완료/실패/취소 후에만 감시가 재개되어야 함

- [ ] CP-15 | 타임라인 / 글로벌 캔버스

- [ ] CP-25 | 자막 품질 검사 / 자동 교정 설계 확인
  - 검사 OFF 기본값에서 기존 자막 생성/편집/저장/멀티클립 흐름이 그대로 유지되어야 함
  - `검사` 버튼 위치와 문구가 현재 에디터 UI 흐름에 어울려야 함
  - `자동 교정` 옵션은 기본 OFF가 맞는지 확인
  - 자동 교정 ON일 때 낮은 점수 구간만 제한적으로 바꾸는 정책이 적절한지 확인
  - 자동 적용 기준인 추천 후보 92점 이상, 기존 대비 10점 이상 개선 기준이 적절한지 확인
  - review VAD ON 상태의 speech pad 0.35초 / min silence 0.8초가 발화 누락 없이 충분히 엄격한지 확인
  - 빠른/균형/정밀 또는 검사 ON 상태의 chunk overlap 0.5초 / 1.5초 / 3.0초 체감 속도와 정확도 균형 확인
  - 무음/VAD 외부 구간 자막을 자동 삭제하지 않고 flag/후보 격리로만 처리하는 정책이 적절한지 확인
  - red/yellow/green/gray 품질 색상이 기존 선택 색상, 화자 색상, 러프컷 색상과 충돌하지 않아야 함
  - 품질 요약 패널 위치가 작업 화면을 방해하지 않아야 함
  - 후보 비교 UI에서 기존 자막/추천 후보/다른 후보/점수/적용 버튼이 이해 가능해야 함
  - 사용자의 수동 수정 내용을 `correction_memory.json`에 자동 저장하는 방식에 동의가 필요한지 확인
  - 프로젝트 JSON에 `quality` / `quality_history`를 얼마나 저장할지 확인
  - 전체 품질 summary를 프로젝트 `editor_state`에 자동 저장할지, 검사 결과 표시용 런타임 값으로만 둘지 확인
  - 한국어 특화 Whisper 모델을 실험 옵션으로 노출하는 방식이 적절한지 확인
  - 한국어 특화 Whisper 실험 모델 로드 실패 시 기존 기본 모델 fallback이 실제 작업을 끊지 않는지 확인
  - stable-ts, whisper-timestamped 같은 외부 패키지를 dependency로 넣지 않고 내부 구현으로 가는 방향 확인

- [ ] CP-26 | PAGE 3-B 러프컷 에디터 UI / 운영 정책 확인
  - 러프컷 에디터 화면에서 상단 `시작` 버튼이 자막 생성이 아니라 러프컷 분석/재분석으로 동작하는 방식이 맞는지 확인
  - 러프컷 전용 LLM 모델 선택, 프롬프트 선택, 프롬프트 편집 위치가 UI 초안 흐름에 어울리는지 확인
  - 중분류 카드 A/B/C와 소분류 A1/A2/B1 row 구조가 실제 리뷰 작업에 충분히 이해 가능한지 확인
  - `임시 확정`, `읽는 중`, `최종 확정`, `확인 필요` 상태 문구와 색상이 현재 앱 스타일과 충돌하지 않는지 확인
  - 오른쪽 실시간 로그와 제목 추천 패널이 작업 화면을 방해하지 않는지 확인
  - 추천 유튜브 제목의 새로고침, 복사, 예상 조회수/점수 표시 방식이 필요한 정보만 보여주는지 확인
  - 하단 자막 세그먼트, 글로벌 세그먼트, waveform, EDL, storyboard tab 구성이 실제 편집 순서와 맞는지 확인

  - 썸네일/hover preview cache를 project-local 또는 temp 경로로 둘지 확인. `dataset/video_preview_cache/`는 건드리지 않음
  - 기존 QTableWidget 러프컷 테이블을 fallback 또는 legacy tab으로 유지하는 방식이 적절한지 확인
  - `roughcut_state` v2 저장 후에도 기존 v1 프로젝트가 열리는지 대표 샘플 프로젝트로 확인
